## Форматы изображений

### Рекомендуемые форматы:

1. **WebP** (основной формат)
   - Лучшее сжатие
   - Поддержка: Chrome, Edge, Firefox, Safari 16+
   - Размер: на 25-35% меньше JPEG

2. **JPEG** (fallback)
   - Универсальная поддержка
   - Хорошее сжатие для фотографий
   - Качество: 75-85%

3. **PNG** (для графики)
   - Без потерь
   - Поддержка прозрачности
   - Используется для логотипов и иконок

4. **AVIF** (будущее)
   - Лучшее сжатие (на 50% меньше JPEG)
   - Поддержка: Chrome 85+, Firefox 93+, Safari 16+

## Оптимизация изображений

### Инструменты для сжатия:

1. **ImageOptim** (Mac)
   - Автоматическое сжатие
   - Поддержка WebP

2. **FileOptimizer** (Windows)
   - Пакетная обработка
   - Множество форматов

3. **TinyPNG/TinyJPG** (онлайн)
   - Простой интерфейс
   - Хорошее качество

4. **ImageMagick** (CLI)
   ```bash
   convert input.jpg -quality 85 -resize 1920x1080 output.jpg
   convert input.jpg -define webp:method=6 output.webp
   ```

## Реализация в коде

### Использование srcset для адаптивных изображений:

```html
<img 
  src="image-640.jpg"
  srcset="
    image-320.jpg 320w,
    image-640.jpg 640w,
    image-1024.jpg 1024w,
    image-1920.jpg 1920w
  "
  sizes="
    (max-width: 640px) 100vw,
    (max-width: 1024px) 90vw,
    80vw
  "
  alt="Описание"
  loading="lazy"
  decoding="async"
/>
```

### Использование picture для разных форматов:

```html
<picture>
  <source srcset="image.avif" type="image/avif" />
  <source srcset="image.webp" type="image/webp" />
  <img src="image.jpg" alt="Описание" loading="lazy" />
</picture>
```

- [ ] Размеры файлов соответствуют рекомендациям
- [ ] Протестировано на разных устройствах (DevTools)

## Тестирование

### В DevTools (F12):

1. Откройте вкладку "Network"
2. Установите throttling: "Slow 4G"
3. Проверьте время загрузки изображений
4. Используйте Device Emulation для разных экранов
